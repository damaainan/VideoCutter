基于需求规格，我们设计一个清晰、可维护的技术架构。总体采用 **分层架构 + 插件式模块** 的思路，UI 与业务逻辑分离，ffmpeg 调用单独封装。

---

## 技术架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                        表现层 (UI)                          │
│  PySide6 Widgets / Layouts / Signals & Slots               │
│  main_window.py  file_list_widget.py  time_input_widget.py │
│  preset_dialog.py  settings_dialog.py  log_widget.py       │
└──────────────────────────┬──────────────────────────────────┘
                           │ (信号/槽 或 直接调用)
┌──────────────────────────▼──────────────────────────────────┐
│                    应用逻辑层 (Core)                        │
│  VideoCutterManager – 任务调度、队列管理、进度跟踪          │
│  PresetManager – 预设增删改查、持久化                      │
│  TimeRangeCalculator – 根据模式计算 -ss / -to 参数          │
│  OverwritePolicy – 文件冲突处理策略                         │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                   基础设施层 (Utils)                        │
│  FFmpegRunner – QProcess 封装，支持取消、错误收集           │
│  FFprobeHelper – 获取视频时长等元数据                       │
│  TimeParser – 时间字符串解析（秒数、HH:MM:SS）              │
│  ConfigManager – QSettings 封装，管理应用配置               │
└─────────────────────────────────────────────────────────────┘
```

## 文件列表与功能说明

### 1. 入口与配置
- **`main.py`**  
  应用程序入口。初始化 `QApplication`，加载全局样式、检测 ffmpeg 可用性，创建并显示主窗口。负责将控制权交给 Qt 事件循环。

- **`config.py`**  
  基于 `QSettings` 的配置管理器。提供统一接口读写用户设置（ffmpeg 路径、默认后缀、文件冲突策略、窗口位置/大小、预设存储路径等）。启动时自动加载，修改后即时保存。

### 2. 表现层（ui 包）
- **`ui/main_window.py`**  
  主窗口类，继承 `QMainWindow`。  
  职责：组装左侧文件列表、右侧控制面板、菜单栏、状态栏；接收拖放事件并分发给文件列表；连接各控件信号到核心逻辑层；作为整个 UI 的总控制器。

- **`ui/file_list_widget.py`**  
  自定义 `QWidget`，内含 `QTableWidget` 或 `QListWidget`（推荐表格，可显示多列）。  
  职责：  
  - 支持文件拖放（`dragEnterEvent`, `dropEvent`），解析路径并去重。  
  - 提供添加、移除、清空按钮。  
  - 显示文件信息（名称、路径、时长、状态），异步更新时长（通过信号接收 `FFprobeHelper` 的结果）。  
  - 右键菜单（移除、重试、打开目录）。

- **`ui/time_input_widget.py`**  
  自定义 `QGroupBox` 或 `QWidget`，管理时间模式切换和时间输入。  
  职责：  
  - 两个 `QRadioButton` 切换“去头去尾”与“绝对起止”模式。  
  - 根据模式动态显示 A/B（去掉开头/结尾）或 Start/End 输入框。  
  - 输入框为 `QLineEdit`，旁边带实时格式化标签（如 `= 00:01:50`）。  
  - 校验输入合法性（红色边框），发射信号 `times_changed` 携带解析后的秒数值及模式类型。

- **`ui/preset_widget.py`**  
  包含预设下拉框、保存/删除预设按钮的小部件，可嵌入控制面板。  
  职责：  
  - 下拉列表显示所有预设名称，选择后发射信号 `preset_applied`，携带预设数据。  
  - “保存当前为预设”按钮：弹出简单对话框输入名称，调用 `PresetManager`。  
  - “删除”按钮确认后删除当前选中预设。

- **`ui/cutter_control_panel.py`**  
  整合 `time_input_widget`、`preset_widget`、开始裁剪按钮、进度条、停止按钮、日志文本框等右侧整体布局。  
  职责：  
  - 集中管理控制流程。  
  - 点击“开始裁剪”时收集文件列表和当前时间参数，传递给 `VideoCutterManager` 启动任务。  
  - 接收任务进度信号，更新进度条和日志。

- **`ui/settings_dialog.py`**  
  模式对话框，继承 `QDialog`。提供 ffmpeg 路径选择、输出后缀修改、文件冲突默认策略选择、是否启用精确模式等选项。修改后立即回写 `ConfigManager`。

- **`ui/preset_dialog.py`**  
  可复用的预设名称输入对话框，用于保存或重命名预设。

### 3. 应用逻辑层（core 包）
- **`core/video_cutter_manager.py`**  
  核心任务管理器，继承 `QObject`。  
  职责：  
  - 接收待处理文件列表和时间参数（解析后的秒数、模式）。  
  - 创建裁剪队列，逐个处理。  
  - 为每个文件构造 `FFmpegRunner` 实例，放入独立线程（`QThread`）执行。  
  - 处理结果、更新文件状态，发出进度信号（`progress_changed`、`file_status_changed`、`all_finished`）。  
  - 提供 `cancel_all()` 方法终止当前及后续任务。

- **`core/time_range_calculator.py`**  
  纯计算类。  
  职责：  
  - 输入视频总时长、模式类型、用户输入的 A/B 或 Start/End（秒数）。  
  - 计算 ffmpeg 所需的 `-ss` 和 `-to` 参数值（秒）。  
  - 验证有效性（开始 < 结束，不超出时长等），返回计算结果或错误信息。

- **`core/preset_manager.py`**  
  管理预设配置。  
  职责：  
  - 从磁盘加载预设 JSON 文件（路径由 `ConfigManager` 提供）。  
  - 提供增删改查接口，返回数据模型（字典列表）。  
  - 支持标记默认预设，启动时自动加载。

- **`core/overwrite_policy.py`**  
  文件冲突处理辅助类。  
  职责：  
  - 根据用户全局设置（询问/自动重命名/跳过）及目标路径，返回实际应使用的输出路径，必要时发射信号要求 UI 弹出询问框（可通过回调实现）。  
  - 自动重命名逻辑：尝试 `_1`、`_2`…直到不存在同名文件。

### 4. 基础设施层（utils 包）
- **`utils/ffmpeg_runner.py`**  
  封装 `QProcess`，运行 ffmpeg 命令。  
  职责：  
  - 接收命令参数列表。  
  - 启动进程，异步捕获 stdout/stderr，完成时发射 `finished` 信号（含返回码和错误输出）。  
  - 提供 `cancel()` 方法向进程发送终止信号。  
  - 不阻塞 UI 线程，适合在 `QThread` 中启动（或直接使用 `QProcess` 的异步 API）。

- **`utils/ffprobe_helper.py`**  
  使用 `QProcess` 同步或异步调用 `ffprobe` 获取媒体信息。  
  职责：  
  - 获取视频时长（`format=duration`），返回浮点秒数。  
  - 可能扩展获取编码、分辨率等信息（当前版本仅需时长）。  
  - 提供批量查询方法，每完成一个发射信号以更新 UI。为不阻塞 UI，应在后台线程中执行（`QThreadPool` + `QRunnable`）。

- **`utils/time_parser.py`**  
  纯函数模块。  
  职责：  
  - `parse_time_input(text: str) -> Optional[float]`：支持纯数字（秒）和 `HH:MM:SS[.sss]` 格式。返回总秒数，失败返回 `None`。  
  - `format_seconds(secs: float) -> str`：将秒数格式化为 `HH:MM:SS` 显示用。

- **`utils/config_manager.py`**  
  `ConfigManager` 类，封装 `QSettings`。  
  职责：  
  - 提供属性或方法读写各项设置，如 `ffmpeg_path`, `suffix`, `conflict_strategy`, `precision_mode` 等。  
  - 配置变更时自动同步到磁盘。

### 5. 资源文件
- **`resources/` 目录**（可选）  
  存放图标、样式 qss 文件等。可通过 Qt 资源系统编译为 `.py` 文件以便打包。

### 6. 辅助文件
- **`requirements.txt`**  
  记录依赖：`PySide6`、`pyinstaller`（打包用）等。

## 主要数据流
1. 用户拖拽或打开文件 → `FileListWidget` 发出信号，`MainWindow` 将文件路径列表交给 `FFprobeHelper` 异步获取时长，并更新列表。
2. 用户设置时间（`TimeInputWidget`） → 实时解析并传给 `TimeRangeCalculator` 预览结果，若合法则启用“开始裁剪”按钮。
3. 用户点击“开始裁剪” → `CutterControlPanel` 收集所有待处理文件路径、当前时间参数、模式、覆盖策略等，调用 `VideoCutterManager.start_batch(...)`。
4. `VideoCutterManager` 遍历文件列表，对每个文件：
   - 通过 `TimeRangeCalculator` 计算 `-ss` 和 `-to`。
   - 通过 `OverwritePolicy` 决定输出路径。
   - 构造 ffmpeg 命令。
   - 创建 `FFmpegRunner` 并运行，连接其 `finished` 信号到内部槽函数。
5. 每个文件完成后，`VideoCutterManager` 更新状态并发射进度信号，UI 更新文件列表和进度条。
6. 全部完成后，汇总信息显示在日志区。

## 并发与线程安全
- 时长获取使用 `QThreadPool` 并行处理。
- ffmpeg 裁剪本身由 `QProcess` 执行（主事件循环驱动），不另开 `QThread`，但可以通过将 `QProcess` 移动到 `QThread` 来避免微小的 UI 卡顿（也可直接在主线程使用，因为 `QProcess` 是异步的）。根据经验，直接在主线程使用 `QProcess` 并连接信号即可，裁剪期间 UI 仍可响应（停止按钮）。
- 所有 UI 更新都必须通过信号在主线程执行，`FFmpegRunner` 内部处理。

---

此架构清晰地分离了界面、业务与基础设施，易于扩展（例如未来增加其他模式或编码选项），同时保持工具的小巧和高效。
