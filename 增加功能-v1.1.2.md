# v1.1.2 新增功能需求梳理

## 1. 功能概述
在现有全局统一时间裁剪基础上，允许用户为文件列表中的**单个视频**设置专属裁剪时间（去头去尾或绝对起止），同时提供一个**默认时间配置**，新添加的文件自动继承默认时间。  
另外，将当前默认时间及部分全局设置持久化到配置文件，便于重启后恢复。

---

## 2. 新增概念与数据模型

### 2.1 文件项数据扩展
每个视频文件条目现在包含以下新增属性（可封装为数据类）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `use_default` | bool | 是否使用默认时间；`True` 时忽略下方自定义时间 |
| `custom_mode` | str/None | `'trim'`（去头去尾）或 `'absolute'`（绝对起止），与全局模式相同 |
| `custom_A` | float/None | 自定义去头时间（秒）或起始时间（模式一/二） |
| `custom_B` | float/None | 自定义去尾时间（秒）或结束时间（模式一/二） |

文件列表中仅展示状态：名称、时长、状态（等待/成功等），现在增加一列“时间来源”，显示“默认”或“自定义”。

### 2.2 默认时间配置
全局默认时间设定，位于右侧控制面板，继续沿用现有时间输入控件（但需增加“设为默认”的语义）。默认时间本身也分两种模式，值可保存。

### 2.3 配置文件扩展
使用本地 JSON 文件（例如 `video_cutter_config.json`）保存：
- **全局默认时间**（模式、A/B 值）
- **预设列表**（可继续用独立预设文件，也可合并）
- **最近使用的文件列表**（可选，本次暂不要求，但可设计）
- **自定义文件时间映射**（路径 -> 时间配置），以便程序重启后恢复每个文件的自定义设置。

保存策略：
- 手动保存：通过菜单“保存配置”或窗口关闭时自动保存。
- 启动时自动加载上次配置，并恢复默认时间和文件列表（如果有）。

---

## 3. UI / 交互调整

### 3.1 右侧控制面板
- 现有“时间设置”区域增加标签，标明为“默认时间（应用于新文件）”。
- 增加一个按钮：“应用到所有文件”，将所有文件的自定义时间重置为当前默认值（即`use_default=True`，清除自定义数据）。

### 3.2 左侧文件列表
- 新增一列“时间”展示概要（例如“默认(去头110s尾50s)”或“自定义(0-120s)”）。
- 右键菜单新增两项：
  - **设置自定义时间**：弹出对话框（复用或简化时间输入控件），允许为选中文件（可多选）单独设置模式和时间。确认后对应文件 `use_default=False`，保存自定义值。
  - **重置为默认时间**：将选中文件的 `use_default` 置回 `True`，清除自定义时间。
- 双击文件条目也可直接打开时间设置对话框。

### 3.3 时间设置对话框（新增）
- 小型模态窗口，包含：
  - 当前文件名显示
  - 模式选择（RadioButton：去头去尾/绝对起止），默认与全局模式相同
  - 时间输入（A/B 或 Start/End），默认填入当前文件的时间值（若有）
  - 两个按钮：“确定”、“取消”
- 支持多文件批量设置：如果多选文件打开对话框，显示的值为空白或“（多个值）”，用户设置后将统一应用到所有选中文件。

### 3.4 状态指示
列表状态列在裁剪过程中已使用，可保持不变。新增一列用图标或颜色表示“默认”或“自定义”。例如绿色圆点表示自定义，灰色表示默认。

---

## 4. 裁剪执行逻辑调整
- 遍历文件列表时，对每个文件检查 `use_default`。
  - 若为 `True`，使用全局默认时间（当前右侧面板值）。
  - 若为 `False`，使用文件自身的 `custom_mode, custom_A, custom_B`。
- 如果自定义时间不完整（例如缺少 B 或模式缺失），可回退到默认值或要求用户修正（可在对话框做校验）。
- 进度和错误处理同前。

---

## 5. 配置保存与恢复

### 5.1 保存内容
配置文件 `config.json` 结构示例：
```json
{
  "default_time": {
    "mode": "trim",
    "A": 110,
    "B": 50
  },
  "presets": [
    {"name": "去片头110s片尾50s", "mode": "trim", "A": 110, "B": 50}
  ],
  "file_custom_times": {
    "C:/videos/demo.mp4": {"mode": "absolute", "A": 30, "B": 120},
    "D:/clips/test.mov": {"mode": "trim", "A": 10, "B": null}
  }
}
```

### 5.2 保存时机
- 手动：菜单栏 -> 文件 -> 保存配置（Ctrl+S）。
- 自动：窗口关闭时触发保存（需在 `closeEvent` 中调用）。
- 程序异常退出时不保证保存，可考虑定时自动保存（暂不实现）。

### 5.3 加载时机
- 程序启动时，在 `main()` 中加载配置文件（若存在），恢复默认时间到 UI，恢复自定义文件时间映射到内存（此时文件列表可能为空，后续添加文件时匹配路径以恢复自定义设置）。
- 添加文件时：检查文件绝对路径是否在 `file_custom_times` 中，若存在则设置该文件的 `use_default=False` 并填充自定义值；否则继承默认设置。
- 如果文件路径已变更（例如移动了文件夹），则匹配失败，自动使用默认时间，不报错。

---

## 6. 兼容性与注意事项
- 老版本预设文件（`presets.json`）仍可保留，但建议将预设也集成进新配置文件，或同时读取两者。
- 如果当前文件之前已有自定义时间，但用户又修改了全局默认时间，文件的自定义状态不受影响，除非用户主动执行“重置为默认”或“应用到所有”。
- 删除文件列表中的文件时，可选择是否同时从配置中移除其自定义时间（简单起见，保存时只保存当前列表中存在且自定义的文件，删除文件即清理）。

---

## 7. 开发优先级与实现建议
1. **数据模型扩展**：修改文件列表项的数据结构。
2. **配置文件读写模块**：新增 `config.json` 读写功能。
3. **UI 调整**：列表增加列、右键菜单、自定义时间对话框。
4. **连接信号**：保证全局默认时间变化时，不会自动覆盖已自定义的文件。
5. **裁剪逻辑更新**：改用文件自身时间优先。
6. **自动加载与保存**：启动恢复和关闭保存。
7. **测试**：多文件混合默认与自定义情况。

这样梳理后，v1.1.2 在保留原有简洁操作的基础上，提供了灵活的“默认+例外”时间管理，同时具备持久化能力。
