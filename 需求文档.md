# 视频极简裁剪工具 – 需求规格说明书

## 1. 项目目标
一款运行于 Windows 的极简视频裁剪工具，专注于**快速去除片头片尾**或**按任意起止点截取**，支持批量视频，保持原始格式，无损流复制裁剪，操作路径最短。

**选定技术栈**：Python 3 + PySide6 + ffmpeg。

---

## 2. 功能模块详述

### 2.1 视频文件导入
- **拖拽添加**：从资源管理器拖拽一个或多个视频文件到程序窗口主区域，解析绝对路径并加入待处理列表。
- **按钮添加**：提供“添加文件”按钮，打开系统多选文件对话框，筛选常见视频格式（*.mp4, *.avi, *.mkv, *.mov, *.wmv 等）。
- **文件列表**：以表格或列表控件展示：
  - 文件名（含扩展名）
  - 文件所在目录
  - **视频时长**（通过 ffprobe 或 ffmpeg 读取，后台线程异步获取，避免阻塞UI）
  - 状态（等待处理、处理中、完成、失败）
- **列表操作**：
  - 右键菜单或按钮：移除选中文件、清空列表。
  - 重复添加同一绝对路径时自动忽略并提示“文件已在列表中”。

### 2.2 时间设置（核心）
提供两种**时间模式**，可通过显眼的 `Tab` 或 `RadioButton` 切换，满足不同习惯。

#### 模式一：去头去尾（推荐默认）
- **去掉开头时长 (A)**：输入要剪掉的起始片段长度。
- **去掉结尾时长 (B)**：输入要剪掉的结尾片段长度。
- **逻辑**：最终保留区间 `[A, 视频总长 - B]`。
- **空值语义**：
  - `A` 为空 → 从头保留。
  - `B` 为空 → 保留到结尾。
- **约束**：`A + B < 视频总时长`，否则提示错误。

#### 模式二：绝对起止时间
- **开始时间 (Start)**：视频保留的起点。
- **结束时间 (End)**：视频保留的终点。
- **逻辑**：保留区间 `[Start, End]`。
- **空值语义**：
  - `Start` 为空 → 默认 `00:00:00`。
  - `End` 为空 → 直到视频结尾。
- **约束**：`Start < End` 且不大于总时长。

#### 时间输入格式
统一支持两种输入形式，自动解析：
1. **`HH:MM:SS`** 或 **`HH:MM:SS.sss`**（毫秒可选）。
2. **纯数字**（视为秒，支持小数，如 `110` 或 `50.5`）。
- 输入框旁提供实时预览（如 `= 00:01:50`），减少迷惑。
- 对非法格式即时红色边框提醒，并禁用“开始裁剪”按钮。

### 2.3 预设配置
针对反复使用的固定去头去尾值（如“去片头 110s 片尾 50s”），提供预设管理。
- **保存预设**：将当前时间模式、A/B 值、模式类型一并存储。
- **应用预设**：通过下拉框选择预设后，自动设置时间模式和数值。
- **管理预设**：重命名、删除预设。
- **存储**：使用 `QSettings` 或本地 `presets.json`，存放于用户 AppData 目录下，保证重开软件保留。
- **默认预设**：可标记一个预设为“启动时自动加载”。

### 2.4 输出设置
- **输出位置**：**始终与原视频同目录**，不可更改（极简原则）。
- **文件命名**：
  - 固定后缀 `_1`，即 `原文件名_1.扩展名`，例如 `video_1.mp4`。
  - 若 `_1` 文件已存在，弹出对话框：
    - **覆盖**：直接替换。
    - **自动重命名**：尝试 `_2`、`_3`... 直到无冲突。
    - **跳过**：保留原文件，本次不处理该视频。
  - 用户还可设置默认策略，避免每次都询问（在设置中配置）。
- **文件格式**：**与原视频容器和编码格式完全一致**，使用 ffmpeg 的 `-c copy` 流复制模式，极速裁剪且无损画质。不做转码。
- **关键帧对齐**：流复制时剪切点会自动对齐到最近的关键帧，因此最终裁剪长度可能有几秒误差。在界面用小字提示“快速模式：剪切点将自动对齐到最近关键帧（误差±几秒）”，并可切换为精确模式（见补充功能）。

### 2.5 裁剪执行
- **批量处理**：对文件列表中的所有文件，依次应用相同的时间参数进行裁剪。
- **执行过程**：
  - 显示整体进度条（当前第 x 个，共 y 个）。
  - 每个文件开始处理时更新其状态为“处理中…”，完成后为“成功”或“失败”，并简要记录（如“输出：xxx_1.mp4”或错误摘要）。
  - **取消按钮**：可随时终止队列，已生成的文件保留，未开始的标记为“已取消”。
- **ffmpeg 命令构造**（以模式一为例）：
  - 获取视频总时长 `D`（已知，列表已缓存）。
  - 若 A 有值，则 `-ss A`；否则不指定 `-ss`。
  - 若 B 有值，计算 `-to` 参数 = `D - B`；否则不指定 `-to`。
  - 最终命令类似：
    ```
    ffmpeg -ss A -i "输入" -to (D-B) -c copy -map 0 -avoid_negative_ts make_zero "输出_1.mp4"
    ```
  - 模式二类似，直接使用 Start/End 填入 `-ss` 和 `-to`。
- **错误处理**：捕获 ffmpeg 的 stderr 输出，若返回码非0，则将错误信息关联到该文件，同时允许继续处理下一个。

### 2.6 结果与通知
- 处理完成后，在日志区汇总成功/失败/跳过数量。
- 提供“打开所在文件夹”快捷按钮（针对选中文件或最后一个成功文件）。
- 支持一键清空已完成文件列表，保留失败项以便重试。

### 2.7 环境检测与设置
- **启动检测**：自动检测系统 PATH 中是否存在 `ffmpeg`，若找不到，弹出设置对话框引导用户手动指定 ffmpeg.exe 路径。同时支持手动点击“测试连接”验证。
- **设置界面**（可选，主窗口菜单栏或设置按钮）：
  - ffmpeg 可执行路径。
  - 输出文件后缀（默认 `_1`，可自定义）。
  - 文件存在时的默认操作（询问 / 自动重命名 / 跳过）。
  - 是否启用“精确模式”（见补充）。
- **持久化**：使用 `QSettings` 保存配置。

## 3. 界面布局建议（PySide6）
主窗口大致分为三个区域：
- **顶部工具栏**：添加文件、清空列表、预设下拉框、保存/删除预设、设置按钮。
- **左侧文件列表**：占据主要面积，带复选框和多选，显示文件名、时长、状态。
- **右侧控制面板**（垂直排列）：
  - 模式切换（两个 RadioButton）。
  - 时间输入区（A/B 或 Start/End），采用 `QTimeEdit` 或普通 `QLineEdit` 组合，加上实时转换标签。
  - 大片醒目的“开始裁剪”按钮。
  - 进度条和日志输出框（只读，可滚动）。
- 底部状态栏：显示 ffmpeg 检测状态、总文件数、预计总输出大小等。

界面支持文件拖放，将 `QListWidget` 或 `QTableView` 设为接受拖放，事件中提取文件路径并过滤视频扩展名。

## 4. 补充功能（把裁剪做到极致的小细节）
在不增加复杂度的前提下，加入以下贴心设计：

1. **智能时长探测**  
   文件加入列表后，后台异步使用 `ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1:nokey=1` 获取精确时长并缓存，失败则标记“未知时长”，但允许强制裁剪（按文件实际长度由 ffmpeg 处理）。

2. **裁剪预览与时长估算**  
   实时计算并显示预计裁剪后时长：`D - A - B`（去头去尾模式）或 `End - Start`（绝对模式），并提示总输出文件个数及预估释放空间（因流复制，体积可粗略按比例估算）。

3. **常用快捷键**  
   - `Ctrl+O` 添加文件。  
   - `Enter` 开始裁剪。  
   - `Escape` 取消当前任务。

4. **精确模式（可选）**  
   提供一个复选框“精确到帧（慢）”，勾选后 ffmpeg 将 `-ss` 放在 `-i` 之后，并移除 `-c copy`，采用重编码至关键帧完全对齐，但速度慢很多。默认不勾选，保持快速流复制。

5. **历史记录**  
   自动记录最近 10 次成功使用的参数（模式、A、B、应用文件数、时间戳），方便回溯。

6. **错误重试**  
   在文件列表中对失败项提供右键“重试”选项，直接重新调用 ffmpeg。

7. **静默覆盖策略**  
   如用户在设置中选了“自动重命名”，则依次尝试 `_1`, `_2` 直到不冲突，展示最终生成的文件名。

## 5. 开发实现要点
- **多线程**：ffmpeg 调用必须放在 `QThread` 或 `QRunnable` 中，避免 UI 冻结；时长探测亦异步进行。
- **命令参数**：为最大限度保留元数据且无损裁剪，统一使用 `-c copy -map 0 -avoid_negative_ts make_zero`。对 mp4 可能需加 `-movflags +faststart`（可设置开关）。
- **取消实现**：通过 `QProcess` 启动 ffmpeg，保存进程对象，取消时调用 `kill()` 或 `terminate()`。
- **打包**：使用 `PyInstaller` 生成单文件 exe，包含 ffmpeg 验证引导；可选项：内嵌便携版 ffmpeg 以彻底免安装。
